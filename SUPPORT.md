# Getting Help

We're here to help! Here's where to find support for CUBRID Labs projects.

## GitHub Discussions

**Use for questions, usage help, and general discussions:**

[GitHub Discussions](https://github.com/orgs/cubrid-labs/discussions)

- **Q&A**: Ask questions about using any CUBRID Labs project
- **Ideas**: Suggest improvements or new features
- **General**: Off-topic discussions and announcements
- **Show & Tell**: Share projects built with CUBRID

Response time: Usually 24–48 hours from community or maintainers.

## GitHub Issues

**Use for bugs and confirmed defects:**

- Report bugs in the specific repository's **Issues** tab
- Include steps to reproduce, expected behavior, and environment details
- Check if a similar issue already exists before opening a new one

Example: [sqlalchemy-cubrid Issues](https://github.com/cubrid-labs/sqlalchemy-cubrid/issues)

## Documentation

Check the README and docs directory in each repository for:
- **API documentation** — Function/method signatures and behavior
- **Examples** — Code samples for common tasks
- **Troubleshooting** — Known issues and solutions
- **DEVELOPMENT.md** — Dev setup and testing (Python projects)

## Response Times

We aim to respond to questions and issues within:
- **Discussions (Q&A)**: 24–48 hours
- **Issues (bugs)**: 2–5 business days
- **Pull requests**: 3–7 business days

## Code of Conduct

All interactions follow our [Code of Conduct](CODE_OF_CONDUCT.md). We're committed to a respectful and inclusive community.
